[23 Jan 2026] menu-service grpc worker OOM 

Menu Workers are getting killed because of OOM, When debugged the root cause through dd profiler, we identified the culprint function which is being used 
in ProcessMenuRealTime RPC which cal oos and min price for items across time intervals along with Adons. 

One of the main reason here is they havent allocated sufficient capacity so go slice dyanamically allocates 2x memory 

Key Takeaway
Always provide capacity when initializing slices if you know the maximum limit they can grow to.

This ensures:

Memory is allocated upfront - single allocation instead of multiple
No dynamic growth - avoids costly reallocations and copies
Better memory locality - reduces fragmentation
Predictable performance - no surprise memory spikes

Go Slice Internals
type slice struct {
    array unsafe.Pointer  // Points to underlying array
    len   int            // Current number of elements
    cap   int            // Capacity of underlying array
}

go


When cap is insufficient:

Go allocates a new larger array (typically 2x)
Copies all elements to the new array
Old array remains until garbage collected
